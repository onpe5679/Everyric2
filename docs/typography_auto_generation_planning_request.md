# Everyric Studio 자동 타이포그래피 생성 UX 기획 요청서

## 배경

Everyric Studio는 음악 가사 싱크 데이터를 After Effects의 편집 가능한 텍스트 레이어로 변환하는 CEP 패널입니다. 목표는 텍스트 애니메이터나 모션 프리셋을 만들지 않고, 오직 텍스트 레이어의 내용, 배치, in/out 시간만 생성해 작업자가 후반 디자인 플러그인이나 수동 편집으로 이어갈 수 있게 하는 것입니다.

현재 자동 생성은 싱크 원자를 의미/호흡 블록으로 묶고 화면 카드에 배치합니다. 하지만 실제 타이포그래피 영상에서 “읽히는 단위”, “화면에 남기는 시간”, “한 화면에 담을 문장량”, “레이어 스택 정리 방식”은 곡 장르, 영상 톤, 편집자 취향에 따라 크게 달라집니다. 따라서 단일 정답 알고리즘이 아니라, 사용자가 의도를 빠르게 고르고 조정할 수 있는 UX 설계가 필요합니다.

## 기획 목표

전문 영상 편집자와 모션 디자이너가 실제 작업 중 믿고 사용할 수 있는 자동 타이포 생성 경험을 설계해주세요.

성공 기준은 다음과 같습니다.

- 사용자가 “한 글자씩 나오는 읽기 어려운 결과”를 피할 수 있어야 합니다.
- 자동 결과가 완성본이 아니라 편집 가능한 초안이라는 점이 명확해야 합니다.
- 의미 단위, 호흡 단위, 박자 단위, 문장 단위 사이를 사용자가 쉽게 조절할 수 있어야 합니다.
- 생성 전 미리보기에서 결과의 레이어 수, 카드 수, 읽기 밀도, 위험 구간을 파악할 수 있어야 합니다.
- 생성 후 AE 타임라인에서 레이어 순서와 마커가 편집자에게 자연스럽게 보여야 합니다.
- 실패, 진행 중, 취소, 부분 성공 상태가 투명하게 보여야 합니다.

## 핵심 질문

1. 자동 타이포 생성의 첫 화면은 어떤 의사결정부터 받아야 하는가?
2. “Readable / Balanced / Rhythmic” 같은 밀도 프리셋은 충분한가, 아니면 영상 장르 기반 프리셋이 필요한가?
3. 가사 분할 기준을 사용자에게 어떤 언어로 보여주는 것이 가장 직관적인가?
4. 사용자는 생성 전에 어느 정도까지 미리보기를 원할까?
5. 실패하거나 마음에 들지 않을 때 재생성 루프는 어떻게 설계해야 하는가?
6. 참고용 마커, 생성 레이어, 이전 생성물 정리 기능은 어디에 있어야 덜 위험하게 느껴지는가?
7. 전문 사용자가 반복 작업 중 가장 많이 만질 옵션과 숨겨도 되는 옵션은 무엇인가?

## 현재 기능 범위

- JSON 싱크 데이터 불러오기
- 현재 컴포지션 오디오와 가사로 로컬 Everyric2 싱크 생성
- 선택 텍스트 레이어의 Source Text만 교체
- 싱크 데이터를 기반으로 텍스트 레이어 자동 생성
- 분할/타이밍 조정값: density, phrase size, words per block, reveal, pause cut, pre-roll, post-roll, blocks per card
- 레이어 스택 순서 선택
- 라인별 참고 마커 추가 및 Everyric 마커 전체 제거
- 환경 확인: Everyric2, RAM, GPU/VRAM

## 명확한 비목표

- 텍스트 애니메이터 생성
- 글자별 모션 생성
- Animation Composer, Shuffle It, GlyphGide 같은 외부 모션 플러그인 대체
- 완성형 디자인 템플릿 자동 제작
- 사용자가 만든 일반 AE 마커나 레이어 삭제

## 산출물 요청

다음 산출물을 요청합니다.

1. 자동 타이포 생성 사용자 여정
2. 주요 화면 와이어프레임
3. 옵션 정보 구조와 기본값 제안
4. 프리셋 체계 제안
5. 생성 전 미리보기 UX
6. 생성 후 타임라인 정리 UX
7. 오류/진행/취소 상태 문구
8. MVP와 차기 버전 범위 분리
9. 실제 곡 테스트 시나리오 5개

## 판단 기준

기획안은 “기능이 많아 보이는 패널”보다 “편집자가 실전에서 덜 불안한 패널”을 우선해야 합니다. 자동 생성 품질보다 더 중요한 것은 결과를 예측하고, 되돌리고, 조정하고, 다시 생성하는 흐름입니다.
